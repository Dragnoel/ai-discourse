# /query — Research a question

## Usage
/query [topic or question]

## Steps
1. Read PROJECT_STATUS.md and wiki/index.md first.
2. Read relevant entity pages (concepts/, claims/, actors/, themes/, constructs/)
   that bear on the query. Read references.bib for citation context.
3. Synthesise only from existing wiki/ pages and references.bib — never from
   training-data knowledge alone. If a claim is unsupported by wiki/ content,
   say so explicitly.
4. Present the answer in the session (not in a wiki page unless asked).
5. Offer to file back as a wiki page:
   - comparison/ if comparing two claims or actors
   - theme/ if characterising a recurring framing
   - constructs/ if defining a measurable construct
   State the proposed page title and type before writing.

## Constraints
- Use only wiki/ content and references.bib as evidence — cite page names
- Do not assert a position on a contested claim
- Do not open entity pages beyond what the query requires
- Do not update PROJECT_STATUS.md unless a new page is filed back
